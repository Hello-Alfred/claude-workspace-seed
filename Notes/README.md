# Notes/ — dated captures

Thoughts, decisions, ideas, and information worth keeping. The things you'd otherwise scribble somewhere and lose.

**Filename:** `YYYY-MM-DD-short-topic.md` (capture date; topic is 2–5 hyphenated lowercase words).

**Never tasks.** Anything actionable is a checklist line in the right project's `tasks.md` — one home per item. A note *about* a decision can mention follow-up work, but the follow-up itself gets a task line.

**Decisions get a why.** When a note records a decision ("decided to", "going with"), Claude asks for the reason and captures it — the *why* is what future-you actually needs.

Claude drafts every note and shows it before writing — dictation rambles; notes shouldn't.
